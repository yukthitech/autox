Resource Param Type

Resource parameters are capable of getting input from various sources. The value for resource parameter indicates the source of content and type of source. For example in below code, the resource parameter is referring to a "file" at path ".\src\test\resources\test-suites\test.properties". 


<ccg:loadProperties contextAttribute="propFile">
	<resource>
		file:.\src\test\resources\test-suites\test.properties
	</resource>
</ccg:loadProperties>

Following are the list of different type of resources that can be specified for resource properties.

Note: If the resource type starts with "raw-", expressions will not be parsed within the resource content. This will take higher preference over external preference. Example: raw-file, raw-res
Resource Type Example Description
file file:.\src\test\resources\test-suites\test.properties Loads the specified file path.
res res:/test.properties Loads the specified path as a classpath resource.
string string:{
   "name": "someName",
   "age": 30
}
 Loads the specified string (on right side) as content of resource.
string property:attr.response Fetches the specified property (in example, attr.response) and coverts into string. Obtained string is then used as resource content.
